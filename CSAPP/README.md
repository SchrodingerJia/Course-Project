# CSAPP 性能实验：卷积运算优化

## 项目简介

本项目是计算机系统（CSAPP）课程的性能优化实验，以图像处理中的卷积运算（旋转和平滑）为例，实践程序性能分析与优化方法。通过实现并对比不同优化策略（如循环分块、循环展开、数据局部性优化等）对程序性能的影响，深入理解现代计算机体系结构（如缓存、流水线）对程序性能的关键作用。

## 项目结构

```
.
├── README.md               # 项目说明文档
├── src/                    # 源代码目录
│   └── kernels.c          # 卷积核（旋转、平滑）的优化实现
├── include/                # 头文件目录（需自行补充或从原项目获取）
│   └── defs.h             # 类型定义（如 pixel, student_t）和宏（如 RIDX）
└── driver/                 # 驱动测试程序目录（需自行补充或从原项目获取）
    ├── driver.c           # 主测试程序

```

**说明**：
- `src/kernels.c` 是本项目的核心文件，包含了需要优化的卷积函数（`rotate` 和 `smooth`）及其基准版本（`naive_rotate`, `naive_smooth`）。
- `include/defs.h` 和 `driver/` 目录下的文件是运行和测试所必需的，但通常由课程实验框架提供。请根据实验指导，将相应的头文件和驱动代码放置到对应目录。

## 核心功能

`src/kernels.c` 实现了两个图像处理操作：

1.  **旋转 (Rotate)**
    - `naive_rotate`: 基础的逐像素旋转实现（作为性能基准）。
    - `rotate`: 优化版本，采用了**分块（Blocking）**和**循环展开（Loop Unrolling）**策略，旨在提高缓存命中率和减少循环开销。

2.  **平滑 (Smooth)**
    - `naive_smooth`: 基础的平滑（模糊）实现，对每个像素计算其3x3邻域的平均值（作为性能基准）。
    - `smooth`: 优化版本，通过**分类处理**（角点、边缘、内部）和**预计算索引**，避免了大量的条件判断和重复计算，显著提升了性能。

## 使用方法

1.  **环境准备**：确保你有C语言编译环境（如gcc）和实验框架（包括`defs.h`和驱动代码）。
2.  **获取框架**：将课程提供的`driver.c`、`defs.h`、`Makefile`等文件分别放入`driver/`和`include/`目录。
3.  **编译与运行**：
    ```bash
    # 进入driver目录
    cd driver
    # 使用提供的Makefile编译
    make
    # 运行测试程序，评估各版本性能
    ./driver
    ```
4.  **性能分析**：程序将输出每个函数（`naive_rotate`, `rotate`, `naive_smooth`, `smooth`）的运行时间（或性能评分），并计算加速比。
5.  **进一步优化**：你可以基于`src/kernels.c`中的代码，尝试其他优化技术（如SIMD指令、多线程等），并注册新的函数版本进行测试。

## 优化技术要点

- **数据局部性**：`rotate`函数通过分块处理，使得对源数据`src`的访问在缓存中更连续，减少缓存失效。
- **减少开销**：`rotate`函数使用循环展开减少循环控制指令；`smooth`函数将像素分类，用移位(`>>`)代替除法，并预计算公共表达式。
- **消除冗余**：`smooth`函数避免了在内部循环中反复计算边界条件和调用辅助函数。
